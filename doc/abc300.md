### ChatGPT の内容

ステップ0：問題の芯だけ掴む

求めたいのは
n = a²·b·c² ≤ N（a,b,c は素数、a<b<c） の個数。

核心の観察：
 • a と c が二乗 → a,c ≤ √N に必ず収まる。
 • c を固定すると、a²·b ≤ ⌊N/c²⌋ =: K を数える問題になる。
 • a を固定すると、b の上限は min(⌊K/a²⌋, c−1)。
あとは (a, b_max] にある素数の個数を数えればよい。

⸻

ステップ1：小さい N を愚直でチェック（検証用）

まずは N ≤ 10⁵ くらいで、全探索 + 素数判定で正解を出すスクリプトを作ります。
（a,c を 2..√N、b を 2..N で回し、a<b<c と a²·b·c²≤N を満たすかチェック）
これは本番では遅いですが、後で作る高速版の答えと照合するために超重要。

やること：
 • エラトステネス（上限は N 程度でOK）
 • 三重ループ（a<c、b<c の枝刈り、積が超えたら break）

これで「答えの正しさチェック装置」を先につくる。

⸻

ステップ2：√N までの素数表（高速版の土台）

本番に向けて、L = ⌊√N⌋ までの素数をふるいで用意。
 • is_prime[0..L]
 • primes（素数リスト）
 • π配列 pi[x] = x以下の素数個数（累積）

これで (a, b_max] に含まれる素数個数 = pi[b_max] − pi[a] を O(1) で取れるようになる。

⸻

ステップ3：二重ループ（c→a）+ πの差分で b を一括カウント
 • 外側：c を素数で昇順に。
 • K = ⌊N / c²⌋（ここは __int128 で安全に）
 • c² > N になったら以降は不可能 → break
 • 内側：a を素数で a < c
 • 早期終了：a²·(a+1) > K なら break（以降の a は全滅）
 • b_max = min( ⌊K/a²⌋, c−1 )
 • 範囲が空なら continue（b_max ≤ a）
 • 追加：add = pi[b_max] − pi[a] を答えへ

ここで b を列挙しないのが勝ち筋。πの差分で一気に足す。

⸻

ステップ4：型まわり（オーバーフロー対策）
 • 乗算の中間値は __int128（(i128)a * a のように片側をキャスト）
 • 最後に long long へ戻すときだけキャスト
 • 入出力は long long。__int128 を直接 cout はできないので注意

⸻

ステップ5：小さい N で照合
 • ステップ1（愚直）とステップ3（高速）で同じ N を複数試し、答えが一致するか確認。
 • 例：N=300, 1000, 5000, 10000 など。

⸻

よくある落とし穴
 • b_max を「素数の上限」と誤解しない → ただの整数の上限。素数個数は π で数える。
 • a²·(a+1) > K の 早期 break を忘れて無駄に回す。
 • c² > N になっても回し続ける（break する）。
 • 乗算のオーバーフロー（__int128 で回避）。

⸻

ミニ版コード（教育用の核心だけ）

// 前提：is_prime[0..L], primes, pi[0..L] は前計算済み
long long solve(long long N) {
    using i128 = __int128_t;
    long long L = (long long) sqrtl((long double)N);

    auto pi_le = [&](long long x)->int {
        if (x < 0) return 0;
        if (x > L) x = L;
        return pi[(int)x];
    };

    long long ans = 0;
    for (int ic = 0; ic < (int)primes.size(); ++ic) {
        long long c = primes[ic];

        i128 cc = (i128)c * c;
        if (cc > (i128)N) break;
        long long K = (long long)((i128)N / cc);      // K = floor(N / c^2)

        for (int ia = 0; ia < ic; ++ia) {            // a < c
            long long a = primes[ia];

            // 最小候補b ~ a+1 を当てはめた下限チェック
            if ((i128)a * a * (a + 1) > (i128)K) break;

            long long bmaxK = (long long)((i128)K / ((i128)a * a));
            long long b_max = min(bmaxK, c - 1);

            if (b_max <= a) continue;

            ans += pi_le(b_max) - pi_le(a);          // a < b ≤ b_max の素数個数
        }
    }
    return ans;
}
